---
title: Quickstart
description: Get the Eventos API running locally and make your first request in minutes.
---

This guide walks you through cloning the Eventos API, configuring your environment, starting the server, and making your first request.

<Steps>
  <Step title="Install prerequisites">
    Make sure you have the following installed before you begin:

    - **Node.js 14.x** — [nodejs.org](https://nodejs.org)
    - **PostgreSQL** — [postgresql.org](https://www.postgresql.org/download/)
    - **git** — [git-scm.com](https://git-scm.com)

    Verify your Node version:

    ```bash
    node --version
    # Should output v14.x.x
    ```
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/eventos-api.git
    cd eventos-api
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    This installs the four runtime dependencies: `express`, `pg`, `dotenv`, and `cors`.
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root:

    ```bash
    touch .env
    ```

    Add the following variables:

    ```bash
    DATABASE_URL=postgresql://user:password@localhost:5432/eventos
    PORT=3000
    ```

    Replace `user`, `password`, and `eventos` with your actual PostgreSQL credentials and database name.

    <Warning>
      Never commit your `.env` file to version control. It contains sensitive database credentials. Add `.env` to your `.gitignore`.
    </Warning>
  </Step>

  <Step title="Set up the database">
    The API queries a table named `Prueba`. Create it in your PostgreSQL database:

    ```sql
    CREATE TABLE Prueba (
      id SERIAL PRIMARY KEY,
      name VARCHAR(255),
      email VARCHAR(255)
    );
    ```

    You can adjust the schema to match your actual data. The API returns all rows and columns from this table.
  </Step>

  <Step title="Start the server">
    ```bash
    node index.js
    ```

    You should see the server listening on the configured port:

    ```
    Server running on port 3000
    ```
  </Step>

  <Step title="Make your first API request">
    With the server running, fetch the participant list:

    ```bash
    curl http://localhost:3000/lista
    ```

    A successful response returns an array of rows from the `Prueba` table:

    ```json
    [
      {
        "id": 1,
        "name": "Ana García",
        "email": "ana@example.com"
      },
      {
        "id": 2,
        "name": "Carlos López",
        "email": "carlos@example.com"
      }
    ]
    ```

    If the table is empty, the response will be an empty array `[]`.

    <Note>
      The API enforces CORS for browser-based requests, allowing only `https://eventosutmach.000webhostapp.com` as the origin. Direct `curl` requests and server-to-server calls are not subject to this restriction, but browsers from any other origin will receive a CORS error.
    </Note>
  </Step>
</Steps>

## Troubleshooting

If you receive a `500 Internal Server Error`, check the following:

- Your `DATABASE_URL` in `.env` is correct and the database is reachable.
- The `Prueba` table exists in the target database.
- PostgreSQL is running and accepting connections.
- If your database requires SSL, ensure the `ssl: true` setting in `index.js` matches your server configuration.
