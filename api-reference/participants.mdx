## GET /lista

Returns all rows from the `Prueba` table as a JSON array. Each element in the array represents a single event participant record.

**Endpoint**

```
GET /lista
```

## Request

This endpoint requires no query parameters, request body, or authentication headers.

| Property | Value |
|----------|-------|
| Method | `GET` |
| Path | `/lista` |
| Query parameters | None |
| Request body | None |
| Authorization | None |

## Response fields

<ResponseField name="id" type="number">
  Unique identifier of the participant record.
</ResponseField>

<ResponseField name="nombre" type="string">
  Participant's full name.
</ResponseField>

<ResponseField name="email" type="string">
  Participant's email address.
</ResponseField>

<ResponseField name="evento" type="string">
  Name of the event the participant registered for.
</ResponseField>

<ResponseField name="fecha" type="string">
  Registration date and time in ISO 8601 format (e.g. `2024-03-15T10:30:00.000Z`).
</ResponseField>

## Examples

<Note>
  Replace `http://localhost:3000` with your production base URL if calling a deployed instance.
</Note>

```bash
curl http://localhost:3000/lista
```

### Error response (500 Internal Server Error)

Returned when the server cannot complete the database query.

```json
{
  "error": "Error en el servidor"
}
```

<Warning>
  A `500` response indicates a server-side database failure. Check the server logs for details. The client cannot resolve this error by changing the request.
</Warning>
